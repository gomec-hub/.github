# Go!Mec Auto Peças | gomec-hub

### Nosso Objetivo!
O **gomec-hub** foi criado para eliminar silos de código e garantir que cada ferramenta desenvolvida internamente seja acessível, documentada e reutilizável por todo o time de tecnologia.

---

### Categorias de Projetos

| Categoria | Descrição |
| :--- | :--- |
| **Apps Principais** | Nossas principais ferramentas e serviços em produção. |
| **Ferramentas Internas** | CLIs, scripts de automação e ferramentas de produtividade. |
| **Libs & SDKs** | Bibliotecas compartilhadas e padrões de integração. |
| **Infraestrutura** | Códigos de Terraform, Docker e configurações de CI/CD. |

### Desenvolvimento Aumentado por IA!
Na Go!Mec, utilizamos ferramentas de IA para acelerar o ciclo de entrega e manter o foco na resolução de problemas complexos:

* **[Lovable](https://lovable.dev/):** Nossa escolha principal para prototipagem rápida e desenvolvimento de interfaces frontend modernas com React.
* **[Windsurf](https://codeium.com/windsurf):** A IDE oficial para desenvolvimento assistido por agentes, garantindo fluxos de trabalho fluidos em C#, JavaScript e Node.js e outras.

### Como Contribuir?
1. **Padrão de Branch:** Sempre crie branches a partir da `main` usando o padrão `feat/nome-da-feature` ou `fix/problema`.
2. **Pull Requests:** Todo código deverá passar por code review. Marque pelo menos um colega do time.
3. **Documentação:** Se criou uma ferramenta nova, garanta que o README dela explique como rodar o comando principal.

---

### 🛡️ Segurança e Padrões
* **2FA Obrigatório:** Todos os membros devem manter a autenticação de dois fatores ativa.
* **Secrets:** Nunca coloque chaves de API no código. Use os *Organization Secrets*.
* **Linting:** Seguimos os padrões de estilo definidos no repositório `gomec-style-guide`.
